## 4. Pain Points Priorizados (Top 3)

### Prioridad 1 — PP-07: Contexto para Runtime v1.1 y Prompt Builder v1.3

**Razón:** Es el único pain point con impacto sistémico. Runtime v1.1 espera `POST /import` (D-INT-01, D-INT-03) al terminar cada ronda de cada agente. Prompt Builder v1.3 espera `GET /context` <500ms (D-MEM-07) para ensamblar prompts con historial. Estos son los dos componentes centrales del ecosistema VTT. Sin Memory Service, ambos operan con funcionalidad degradada — no es un inconveniente, es capacidad faltante en infraestructura ya diseñada.

**Lo que Memory Service R1 resuelve:** POST /import persiste cada conversación automáticamente. GET /context entrega contexto estructurado en <500ms al Prompt Builder. El contrato de integración está definido en el ADDENDUM v1.1.

### Prioridad 2 — PP-01: Arranque en frío de cada agente

**Razón:** Ocurre en cada invocación de cada agente. El overhead de compensación manual (TL escribiendo secciones de contexto en cada assignment, archivos LOGIC.md como parches de memoria) se traduce en más tokens de entrada y más tiempo humano. Este es el pain point que más directamente experimenta el equipo diariamente.

**Lo que Memory Service R1 resuelve:** GET /context filtra por projectId (D-MEM-39), devuelve conversaciones relevantes ordenadas por startedAt, y el Prompt Builder las integra en el prompt del agente. El arranque en frío se reemplaza por arranque con contexto.

### Prioridad 3 — PP-02: Costos de tokens sin reporte consolidado

**Razón:** Sin visibilidad de costos, el PM no puede gestionar el presupuesto de operación de agentes. Los datos existen fragmentados (usage.jsonl por agente) pero no están centralizados ni agregados. Con corridas individuales de hasta $2.53 USD y múltiples agentes por proyecto, el costo acumulado es significativo pero invisible.

**Lo que Memory Service R1 resuelve:** ConversationUsage persiste tokens + USD por conversación (SPEC §4.1). Los endpoints GET /projects/:id/cost-report y GET /agents/:id/cost-report agregan estos datos por proyecto y por agente con breakdown por tipo de token.
