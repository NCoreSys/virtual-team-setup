# Spec: Usage Tracker Module

## Qué es
Módulo reutilizable para capturar y persistir métricas de uso (tokens, costo, duración) de cada corrida de agente.

---

## 1. De dónde vienen los datos

El SDK (`claude_agent_sdk`) entrega los datos al final de cada corrida en el mensaje `ResultMessage`.

```python
elif isinstance(message, ResultMessage):
    total_cost   = getattr(message, "total_cost_usd", 0.0)
    usage        = getattr(message, "usage", None)       # dict con tokens
    duration_ms  = getattr(message, "duration_ms", 0)
    num_turns    = getattr(message, "num_turns", 0)
    is_error     = getattr(message, "is_error", False)
    session_id   = getattr(message, "session_id", None)
```

### Campos del objeto `usage` (viene del SDK):

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `input_tokens` | int | Tokens de entrada del modelo |
| `output_tokens` | int | Tokens generados por el modelo |
| `cache_creation_input_tokens` | int | Tokens usados para crear cache |
| `cache_read_input_tokens` | int | Tokens leídos desde cache (más baratos) |
| `server_tool_use` | dict | `web_search_requests`, `web_fetch_requests` |
| `service_tier` | str | Ej: `"standard"` |
| `speed` | str | Ej: `"standard"` |

> **Nota**: `usage` llega como dict (ya serializado por `safe_jsonable`). Si llega como objeto, usar `getattr`.

---

## 2. Schema del archivo `usage.jsonl`

Una línea JSON por corrida. Append — nunca se sobreescribe.

```json
{
  "timestamp":                   "2026-03-16T07:02:14.756273+00:00",
  "agentId":                     "fe-agent",
  "runId":                       "experimental_fe_001",
  "taskId":                      "FE-001",
  "sessionId":                   "4070041e-bbd2-4d99-9436-ba5730d09a12",
  "costUsd":                     0.82,
  "durationMs":                  120000,
  "numTurns":                    6,
  "isError":                     false,
  "usage": {
    "input_tokens":              1200,
    "output_tokens":             850,
    "cache_creation_input_tokens": 4500,
    "cache_read_input_tokens":   9800
  }
}
```

### Campos requeridos vs opcionales:

| Campo | Requerido | Fuente |
|-------|-----------|--------|
| `timestamp` | ✅ | `datetime.now(timezone.utc).isoformat()` |
| `agentId` | ✅ | Constante del script (`"fe-agent"`, `"agent2"`) |
| `runId` | ✅ | Constante del script |
| `taskId` | ✅ | Constante del script |
| `sessionId` | ✅ | `ResultMessage.session_id` |
| `costUsd` | ✅ | `ResultMessage.total_cost_usd` |
| `durationMs` | ✅ | `ResultMessage.duration_ms` |
| `numTurns` | ✅ | `ResultMessage.num_turns` |
| `isError` | ✅ | `ResultMessage.is_error` |
| `usage` | ✅ | `ResultMessage.usage` (solo los 4 campos de tokens) |

---

## 3. Ubicación del archivo

```
experimental/agents-state/{agentId}/usage.jsonl
```

Ejemplos:
- `experimental/agents-state/fe-agent/usage.jsonl`
- `experimental/agents-state/agent2/usage.jsonl`

El módulo recibe `agent_id` y resuelve el path automáticamente.

---

## 4. Interfaz del módulo

Archivo: `experimental/agent-runtime/usage_tracker.py`

```python
# Uso esperado desde cualquier agent-runtime script:

from usage_tracker import append_usage

append_usage(
    agent_id="fe-agent",
    run_id=RUN_ID,
    task_id=TASK_ID,
    session_id=session_id,
    cost_usd=total_cost,
    duration_ms=duration_ms,
    num_turns=num_turns,
    is_error=is_error,
    usage=usage,          # dict directo del SDK
)
```

La función:
1. Construye el dict con el schema del punto 2
2. Extrae solo los 4 campos de tokens del dict `usage`
3. Hace append al archivo `agents-state/{agent_id}/usage.jsonl`
4. No lanza excepción si el archivo no existe — lo crea

---

## 5. Cómo leer el archivo

### Todas las corridas de un agente:
```python
import json
from pathlib import Path

def read_usage(agent_id: str, project_root: Path) -> list[dict]:
    path = project_root / "experimental" / "agents-state" / agent_id / "usage.jsonl"
    if not path.exists():
        return []
    return [json.loads(line) for line in path.read_text(encoding="utf-8").splitlines() if line.strip()]
```

### Última corrida:
```python
entries = read_usage("fe-agent", PROJECT_ROOT)
last = entries[-1] if entries else None
```

### Total acumulado de costo por agente:
```python
entries = read_usage("fe-agent", PROJECT_ROOT)
total = sum(e.get("costUsd", 0) for e in entries)
```

### Total de tokens por corrida:
```python
for entry in entries:
    u = entry.get("usage", {})
    total_tokens = (u.get("input_tokens") or 0) + (u.get("output_tokens") or 0)
    print(f"{entry['runId']}: {total_tokens} tokens | ${entry['costUsd']:.4f}")
```

## 9. Extensión futura: Hook Manager (agentes manuales)

> **Estado**: Planeado — no implementar hasta que exista el runtime del Hook Manager.

### Problema

Los agentes corridos vía **Claude Code CLI** (sesiones manuales) no pasan por `ResultMessage`. El SDK no está en el loop — el runtime es el propio Claude Code. Por lo tanto `append_usage()` nunca se llama y la sesión queda sin registro en `usage.jsonl`.

Ejemplos de sesiones que hoy no se rastrean:
- fe-agent corrido directamente desde Claude Code
- Sesiones de revisión o debugging ad-hoc
- Cualquier agente que el TL lanza manualmente sin script Python

### Solución planeada: Hook Manager

Claude Code expone **hooks** que se ejecutan en eventos del ciclo de vida de la sesión. El Hook Manager interceptará el evento de fin de sesión y llamará al mismo `append_usage()`.

```
Evento: PostSessionEnd (o equivalente en Claude Code hooks)
    ↓
Hook Manager lee los datos de la sesión terminada
    ↓
Llama append_usage() con source="cli"
    ↓
Escribe en agents-state/{agentId}/usage.jsonl
```

### Schema extendido para corridas manuales

Mismo schema base del punto 2, con dos campos adicionales:

```json
{
  "timestamp":    "2026-03-27T10:00:00+00:00",
  "agentId":      "fe-agent",
  "runId":        "cli-fe-agent-20260327-100000",
  "taskId":       "FE-004",
  "sessionId":    "session-id-de-claude-code",
  "costUsd":      null,
  "durationMs":   null,
  "numTurns":     null,
  "isError":      false,
  "source":       "cli",
  "usage": {
    "input_tokens":              null,
    "output_tokens":             null,
    "cache_creation_input_tokens": null,
    "cache_read_input_tokens":   null
  }
}
```

> Campos `null` indican que el Hook Manager no pudo obtener ese dato desde CLI. Se populan cuando el hook tenga acceso a los datos de la sesión.

### Campo `source` (nuevo)

| Valor | Significado |
|-------|-------------|
| `"sdk"` | Corrida programática vía `agent-runtime_*.py` (default) |
| `"cli"` | Corrida manual vía Claude Code CLI, capturada por Hook Manager |

### Origen del costo según `source`

| Campo | `source: "sdk"` | `source: "cli"` |
|-------|-----------------|-----------------|
| `costUsd` | ✅ Real — viene de `ResultMessage.total_cost_usd` | ⚠️ `null` — Claude Code CLI no lo expone todavía |
| `usage.input_tokens` | ✅ Real | ⚠️ `null` |
| `usage.output_tokens` | ✅ Real | ⚠️ `null` |
| `usage.cache_*_tokens` | ✅ Real | ⚠️ `null` |

> Cuando el Hook Manager pueda leer los datos de sesión de Claude Code, los campos `null` se popolarán y `source: "cli"` tendrá el mismo nivel de detalle que `source: "sdk"`.

**Implicación para reportes de costo acumulado:**
Al sumar costos totales por agente, filtrar por `source` para saber qué porcentaje del costo está medido vs estimado:

```python
sdk_cost = sum(e["costUsd"] for e in entries if e.get("source", "sdk") == "sdk" and e["costUsd"])
cli_cost_unknown = sum(1 for e in entries if e.get("source") == "cli")
# cli_cost_unknown = N sesiones sin costo medido
```

### Interfaz esperada del Hook Manager

```python
# Cuando el Hook Manager esté construido, llamará:

from usage_tracker import append_usage

append_usage(
    agent_id=agent_id,       # resuelto desde el contexto de la sesión CLI
    run_id=generated_run_id, # generado por el hook: "cli-{agentId}-{timestamp}"
    task_id=task_id,         # tomado del contexto si está disponible, sino None
    session_id=session_id,   # disponible desde Claude Code
    cost_usd=None,           # Claude Code CLI no expone este dato aún
    duration_ms=None,        # ídem
    num_turns=None,          # ídem
    is_error=is_error,
    usage=None,              # ídem — se deja null hasta que el hook pueda leerlo
    source="cli",            # distingue el origen
)
```

### Lo que el Hook Manager necesita del Usage Tracker

- `append_usage()` debe aceptar `source` como parámetro opcional (default `"sdk"`)
- `append_usage()` debe tolerar todos los campos numéricos en `None`
- El schema del archivo no cambia — solo se agregan `null` donde no hay datos
