## 1. PROBLEMA DETECTADO

Se han identificado tareas en producción con:
- **Tiempo real = 0 minutos** (o null)
- **Tiempo real < 5 minutos** (sospechosamente bajo para tareas de desarrollo)

Esto indica:
1. **Errores de tracking**: Las transiciones de estado no se registran correctamente
2. **Errores de datos**: Los timestamps están mal o faltan
3. **Errores de proceso**: Tareas cerradas sin trabajo real (bypass del flujo)

---

## 2. OBJETIVO

Implementar **métricas operativas de detección de anomalías** que permitan:
- Identificar datos sospechosos en tiempo real
- Alertar sobre patrones anómalos
- Proveer información para debugging del sistema de tracking
- Mejorar la confiabilidad de las métricas de negocio

---

## 3. MÉTRICAS OPERATIVAS PROPUESTAS

### 3.1 Categoría: Anomalías de Tiempo

| ID | Métrica | Descripción | Umbral Default | Severidad |
|----|---------|-------------|----------------|-----------|
| **OP-TIME-01** | Zero Duration Tasks | Tareas completadas/aprobadas con Cycle Time = 0 | 0 mins | 🔴 Crítico |
| **OP-TIME-02** | Suspiciously Fast Tasks | Tareas completadas en < 5 minutos | < 5 mins | 🟡 Warning |
| **OP-TIME-03** | Missing First In-Progress | Tareas completadas sin `firstInProgressAt` | null | 🔴 Crítico |
| **OP-TIME-04** | Instant Approval | Tareas aprobadas en < 1 minuto después de complete | < 1 min | 🟡 Warning |
| **OP-TIME-05** | Negative Duration | Tareas con `lastCompleteAt` < `firstInProgressAt` | negativo | 🔴 Crítico |

### 3.2 Categoría: Anomalías de Estado

| ID | Métrica | Descripción | Condición | Severidad |
|----|---------|-------------|-----------|-----------|
| **OP-STATE-01** | Orphan Completions | Tareas en `task_completed` sin transición previa a `task_in_progress` | Sin historial | 🟡 Warning |
| **OP-STATE-02** | Skip States | Tareas que saltaron estados (ej: `task_defined` → `task_completed`) | Gap en flujo | 🟡 Warning |
| **OP-STATE-03** | Zombie WIP | Tareas en `task_in_progress` por > 30 días sin actividad | > 30 días | 🟡 Warning |
| **OP-STATE-04** | Phantom Rejections | Tareas con `rejectedCount > 0` pero sin transición a `task_rejected` en historial | Inconsistencia | 🔴 Crítico |

### 3.3 Categoría: Anomalías de Estimación

| ID | Métrica | Descripción | Umbral | Severidad |
|----|---------|-------------|--------|-----------|
| **OP-EST-01** | Zero Estimate Tasks | Tareas completadas con `estimatedHours = 0` o null | 0/null | 🟡 Warning |
| **OP-EST-02** | Extreme Overrun | Tareas con tiempo real > 10x estimado | > 1000% | 🟡 Warning |
| **OP-EST-03** | Extreme Underrun | Tareas completadas en < 5% del tiempo estimado | < 5% | 🟡 Warning |

### 3.4 Categoría: Anomalías de Datos

| ID | Métrica | Descripción | Condición | Severidad |
|----|---------|-------------|-----------|-----------|
| **OP-DATA-01** | Missing Timestamps | Tareas aprobadas sin `approvedAt` en cache | null | 🔴 Crítico |
| **OP-DATA-02** | Future Timestamps | Tareas con timestamps en el futuro | > now() | 🔴 Crítico |
| **OP-DATA-03** | Duplicate Transitions | Múltiples transiciones idénticas en historial | duplicados | 🟡 Warning |
| **OP-DATA-04** | Orphan History | Registros en `TaskStatusHistory` para tareas que no existen | FK roto | 🔴 Crítico |

### 4.3 Queries de Detección

#### OP-TIME-01: Zero Duration Tasks
```sql
SELECT t.id, t."taskId", t."firstInProgressAt", t."lastCompleteAt",
       EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) / 60 as duration_mins
FROM tasks t
WHERE t.status IN ('task_completed', 'task_approved', 'task_closed')
  AND t."firstInProgressAt" IS NOT NULL
  AND t."lastCompleteAt" IS NOT NULL
  AND t."lastCompleteAt" = t."firstInProgressAt"
  AND t."projectId" = :projectId;
```

#### OP-TIME-02: Suspiciously Fast Tasks
```sql
SELECT t.id, t."taskId", t."firstInProgressAt", t."lastCompleteAt",
       EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) / 60 as duration_mins
FROM tasks t
WHERE t.status IN ('task_completed', 'task_approved', 'task_closed')
  AND t."firstInProgressAt" IS NOT NULL
  AND t."lastCompleteAt" IS NOT NULL
  AND EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) < 300  -- < 5 mins
  AND EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) > 0
  AND t."projectId" = :projectId;
```

#### OP-TIME-03: Missing First In-Progress
```sql
SELECT t.id, t."taskId", t.status, t."firstInProgressAt"
FROM tasks t
WHERE t.status IN ('task_completed', 'task_approved', 'task_closed')
  AND t."firstInProgressAt" IS NULL
  AND t."projectId" = :projectId;
```

#### OP-STATE-01: Orphan Completions
```sql
SELECT t.id, t."taskId"
FROM tasks t
WHERE t.status IN ('task_completed', 'task_approved', 'task_closed')
  AND NOT EXISTS (
    SELECT 1 FROM task_status_history h
    WHERE h."taskId" = t.id
      AND h."toStatus" = 'task_in_progress'
  )
  AND t."projectId" = :projectId;
```

## 5. HEALTH SCORE

Fórmula propuesta para calcular un "score de salud" de los datos:

```
healthScore = 100 - (criticalCount * 10) - (warningCount * 2)
```

| Score | Interpretación |
|-------|----------------|
| 90-100 | ✅ Excelente — datos confiables |
| 70-89 | 🟡 Aceptable — revisar warnings |
| 50-69 | 🟠 Preocupante — investigar |
| < 50 | 🔴 Crítico — datos no confiables |

## 10. DATOS DE EJEMPLO PARA INVESTIGACIÓN

Para investigar el problema actual, necesitamos ejecutar:

```sql
-- Tareas con tiempo real = 0 o sospechosamente bajo
SELECT 
    t."taskId",
    t.status,
    t."firstInProgressAt",
    t."lastCompleteAt",
    t."approvedAt",
    EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) / 60 as cycle_time_mins,
    t."estimatedHours"
FROM tasks t
WHERE t.status IN ('task_completed', 'task_approved', 'task_closed')
  AND (
    t."firstInProgressAt" IS NULL
    OR t."lastCompleteAt" IS NULL
    OR t."lastCompleteAt" <= t."firstInProgressAt"
    OR EXTRACT(EPOCH FROM (t."lastCompleteAt" - t."firstInProgressAt")) < 300
  )
ORDER BY t."updatedAt" DESC
LIMIT 50;
```
