# agent_session.py — Code Logic

## Archivo
`memory/agent_session.py`

## Propósito
Orquestador de conversación para el agente PM Revisor. Gestiona el estado de la conversación entre turns (historial), construye el system prompt modular (base + fase), y llama al CLI de Claude vía `claude_agent_sdk` sin necesidad de API key.

---

## Dependencias
- `claude_agent_sdk` — SDK Python que lanza el CLI de Claude como subprocess
- `requests` — para llamadas al rt-api (workflows HTTP)
- `anyio` / `asyncio` — gestión de event loop (Windows requiere ProactorEventLoop)

---

## Clases y estructuras de datos

### `AgentStep`
Representa un paso individual del agente durante un turn:
- `kind` — tipo: `"text"`, `"thinking"`, `"tool_use"`, `"tool_result"`
- `tool` — nombre de herramienta (`"Read"`, `"Edit"`, `"Bash"`, etc.) o vacío
- `title` — título legible para mostrar en UI
- `content` — contenido principal (texto, output de tool, etc.)
- `metadata` — datos extra según tipo (file_path, old_string, command, etc.)

### `TurnResult`
Resultado completo de un turn:
- `text` — respuesta final en texto plano (para historial)
- `cost_usd` — costo del turn en USD
- `num_turns` — turns internos del SDK (cada tool call = 1 turn)
- `steps` — lista de `AgentStep` para vista detallada en UI
- `messages_raw` — entries JSONL crudas del SDK (para debug)

### `AgentSession`
Clase principal. Gestiona una conversación completa.

## Flujo de una llamada `send()`

```
send(user_message, phase)
  │
  ├─ Detectar si hay event loop corriendo (Streamlit)
  │    ├─ Sí → ejecutar en ThreadPoolExecutor con asyncio.run()
  │    └─ No → asyncio.run() directo
  │
  └─ _send_async(user_message, phase)
       │
       ├─ 1. Construir system_prompt
       │      base_prompt + "\n\n" + fase_X.md (si hay fase activa)
       │
       ├─ 2. Construir prompt
       │      "--- Conversación previa ---"
       │      [historial de turns anteriores]
       │      "--- Fin conversación previa ---"
       │      [mensaje actual del usuario]
       │
       ├─ 3. Llamar claude_agent_sdk.query()
       │      ClaudeAgentOptions(system_prompt, max_turns=1, permission_mode="default")
       │      Iterar async sobre mensajes del SDK
       │
       ├─ 4. Extraer de cada mensaje:
       │      AssistantMessage → texto de respuesta
       │      ResultMessage    → cost_usd, num_turns
       │
       ├─ 5. Parsear steps → _parse_steps(raw_entries)
       │
       ├─ 6. Acumular en historial self.messages
       │
       └─ 7. Guardar sesión → _save()
```

---

## Función `_parse_steps(raw_entries)`

Convierte los mensajes crudos del SDK en `AgentStep` estructurados para la UI.

**Lógica:**
- Itera sobre cada entry del SDK
- `AssistantMessage.content` puede tener bloques tipo: `text`, `thinking`, `tool_use`
- `ToolResultMessage` contiene el resultado de cada tool call
- Usa un mapa `tool_id → step_index` para emparejar `tool_use` con su `tool_result` correspondiente y actualizar el `.content` del step

---

## Gestión de historial

El LLM es stateless — no tiene memoria entre llamadas. El orquestador (`AgentSession`) mantiene el historial y lo inyecta en cada prompt como texto plano formateado:

```
USUARIO:
[mensaje anterior del usuario]

ASISTENTE:
[respuesta anterior del agente]

USUARIO:
[mensaje actual]
```

Esto permite reanudar conversaciones cargando el `session_*.json`.

---

## Gestión del event loop (Windows)

Dos fixes necesarios para Windows + Streamlit:

1. `asyncio.set_event_loop_policy(asyncio.WindowsProactorEventLoopPolicy())` — `SelectorEventLoop` (default en algunos contextos) no soporta subprocesos. `ProactorEventLoop` sí.
2. `send()` detecta si ya hay un loop corriendo (Streamlit) y en ese caso crea un nuevo loop en un thread separado via `ThreadPoolExecutor`.

## Decisiones de diseño

| Decisión | Razón |
|----------|-------|
| Sin API key | Usuario usa suscripción de Claude CLI, no API comercial |
| `max_turns=1` por defecto | El PM Revisor no necesita tool use propio — solo texto. Si se sube, puede usar herramientas |
| Historial como texto en el prompt | claude_agent_sdk no soporta historial nativo de mensajes — el orquestador lo simula |
| `os.environ.pop("CLAUDECODE")` | Evita error "Cannot launch inside another Claude Code session" cuando Streamlit se inicia desde una terminal de Claude Code |
| Módulos de fase separados | Optimización de tokens: base (~800 tokens siempre) + fase (~300-500 tokens solo cuando aplica) |
