# app.py — Code Logic

## Archivo
`memory/app.py`

## Propósito
Interfaz gráfica de chat para el agente PM Revisor. Construida con Streamlit. Permite interactuar con el agente, adjuntar documentos, seleccionar fase Scrum activa, y ver el detalle de cada paso del agente (tool calls, thinking, diffs) similar a Claude Code.

---

## Dependencias
- `streamlit` — framework de UI
- `memory.agent_session` — `AgentSession`, `AgentStep` (orquestador de conversación)
- `requests` — para listar workflows del rt-api desde el sidebar

---

## Estructura de la app

```
app.py
  │
  ├─ render_steps()        ← renderiza pasos detallados del agente
  ├─ _guess_lang()         ← detecta lenguaje por extensión de archivo
  │
  ├─ Sidebar
  │    ├─ Selector de fase (Fase 2, 3B, 5)
  │    ├─ Ver prompt de fase (expander)
  │    ├─ Nueva sesión / Cargar sesión
  │    ├─ Adjuntar documentos (file_uploader)
  │    ├─ rt-api Workflows (listar)
  │    ├─ Checkbox "Vista detallada"
  │    └─ Info: session_id, turns, fase, system prompt
  │
  ├─ Inicializar estado (session_id, messages, turn_results)
  │
  ├─ Historial de chat
  │    ├─ Usuario: markdown del mensaje
  │    └─ Asistente: render_steps() o markdown + métricas (turns, costo, chars)
  │
  └─ Input de chat
       └─ Enviar mensaje → AgentSession.send() → renderizar resultado
```

---

## Estado de Streamlit (`st.session_state`)

| Clave | Tipo | Contenido |
|-------|------|-----------|
| `session_id` | str | ID de sesión actual (timestamp) |
| `messages` | list | Historial de mensajes `{role, content}` |
| `turn_results` | list | Métricas + steps por cada turn `{num_turns, cost_usd, chars, steps}` |

---

## Flujo de envío de mensaje

```
Usuario escribe mensaje
  │
  ├─ Adjuntar archivos (si los hay)
  │    └─ Inyectar contenido de cada archivo en el mensaje:
  │         "--- Documentos adjuntos ---\n### Archivo: X\n```\ncontenido\n```"
  │
  ├─ Mostrar mensaje de usuario en chat (solo texto + nombres de archivos)
  │
  ├─ Crear AgentSession con historial previo restaurado
  │
  ├─ AgentSession.send(full_message, phase=active_phase)
  │
  ├─ Si show_details y hay steps → render_steps(result.steps)
  │   Sino → st.markdown(result.text)
  │
  ├─ Mostrar métricas (SDK turns, costo USD, chars)
  │
  └─ Guardar en session_state:
       messages.append({role: "assistant", content: result.text})
       turn_results.append({num_turns, cost_usd, chars, steps: serialized})
```

## Función `render_steps(steps)`

Renderiza cada `AgentStep` con UI específica según tipo:

| Kind / Tool | Renderizado |
|-------------|-------------|
| `text` | `st.markdown()` — respuesta principal |
| `thinking` | Expander colapsado con texto en itálica |
| `tool_use / Read` | Expander con código resaltado por lenguaje |
| `tool_use / Edit` | Expander expandido con 2 columnas: Antes / Después |
| `tool_use / Write` | Expander con contenido del archivo |
| `tool_use / Bash` | Expander con output estilo terminal |
| `tool_use / Glob,Grep` | Expander con resultados de búsqueda |
| `tool_use / otros` | Expander genérico con JSON de metadata |
| `tool_result` | Expander con texto del resultado |

---

## Función `_guess_lang(filepath)`

Detecta el lenguaje para `st.code()` basado en extensión del archivo. Soporta: `.py`, `.js`, `.ts`, `.tsx`, `.jsx`, `.json`, `.md`, `.yaml`, `.html`, `.css`, `.sql`, `.sh`.

---

## Inyección de documentos adjuntos

Los archivos subidos via `st.file_uploader` se inyectan en el cuerpo del mensaje como texto plano formateado. El agente los recibe como parte del prompt, no como adjuntos binarios.

El mensaje mostrado en el chat al usuario solo indica los nombres de archivos adjuntos (no el contenido completo), para mantener el historial legible.

---

## Vista detallada vs texto plano

Checkbox en sidebar "Vista detallada (tool calls)":
- **Activado**: muestra cada paso del agente (thinking, tool calls, diffs, etc.)
- **Desactivado**: solo muestra el texto final de la respuesta

Al recargar una sesión guardada, los steps se reconstruyen desde `turn_results[i].steps` (serializado como lista de dicts).

---

## Decisiones de diseño

| Decisión | Razón |
|----------|-------|
| Streamlit en lugar de Flask/FastAPI+React | Más rápido de construir, suficiente para uso interno del equipo |
| `max_turns=1` en AgentSession | El PM Revisor responde en un solo turno de LLM — no necesita ciclos de tool use propios |
| Historial restaurado desde session_state | AgentSession es stateless entre reruns de Streamlit — se reconstruye con el historial previo |
| Steps serializados en turn_results | Permite re-renderizar la vista detallada al recargar la página sin releer el JSONL |
| Documentos como texto inline | claude_agent_sdk no soporta adjuntos binarios — el contenido se inyecta en el prompt |
