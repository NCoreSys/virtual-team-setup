## DoD — Backend (Endpoints BE/DB)

Una tarea de backend está "Done" cuando cumple TODOS estos criterios:

- [ ] Código compila sin errores TypeScript (`tsc --noEmit` pasa)
- [ ] Validación Zod implementada para el endpoint (schemas según 2.5.3)
- [ ] Tests unitarios pasan (`npm test` sin fallos)
- [ ] Endpoint probado localmente con curl o Postman (happy path + error path)
- [ ] Swagger/JSDoc actualizado para el endpoint
- [ ] Máquina de estados respetada: ninguna transición prohibida (ver 2.5.6)
- [ ] Idempotencia verificada: reimport retorna ALREADY_INDEXED sin duplicados
- [ ] Code Logic (.LOGIC.md) creado o actualizado
- [ ] Development Log creado con decisiones tomadas
- [ ] PR creado con descripción que referencia la tarea VTT
- [ ] No hay `console.log` de debug en el código
- [ ] No hay TODOs sin resolver en el código

**Criterios adicionales por feature:**

| Feature | Criterio adicional |
|---------|-------------------|
| Import | Archivo JSONL escrito en /storage/, contentPreview calculado, clasificación ejecutada |
| Context | Latencia p95 <500ms verificada con test de carga local, fail-fast 504 implementado |
| Cost | ConversationUsage.totalCostUsd correcto para SDK, null para adapters sin costo |
| Cleanup | Cron registrado y ejecutando cada 5 min, retryCount/MAX_RETRIES implementado |

## DoD — Frontend (Componentes FE)

- [ ] Componente renderiza sin errores en browser (Chrome latest)
