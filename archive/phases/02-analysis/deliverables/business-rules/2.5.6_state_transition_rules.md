## 1. Estados del Sistema

| Status | Código catálogo | Significado |
|--------|----------------|-------------|
| **PENDING** | `PENDING` | Conversación registrada en BD, esperando procesamiento |
| **PROCESSING** | `PROCESSING` | Escritura en storage, parseo y clasificación en curso |
| **IMPORTED** | `IMPORTED` | Import completado exitosamente — conversación disponible para consulta |
| **ERROR** | `ERROR` | Import falló definitivamente — solo cleanup cron produce este estado |

**Ref:** D-MEM-13

                    │       POST /import recibido           │
                    │       POST /import-review recibido    │
                    │       POST /upload recibido           │

                                    │

                         status → PROCESSING)                        │
                                  │                                  │

                             │PROCESSING│                           │

                                           │                      │  │
                                           │  PROCESSING +        │  │

## 3. Reglas de Transición

| # | Desde | Hacia | Trigger | Condición | Ref. |
|---|-------|-------|---------|-----------|------|
| ST-01 | — | PENDING | POST /import, /import-review, /upload | Validación OK, idempotencia OK | D-MEM-13 |
| ST-02 | PENDING | PROCESSING | Inicio de procesamiento | Automático al iniciar escritura/parseo | D-MEM-34 |
| ST-03 | PROCESSING | IMPORTED | Procesamiento exitoso | Escritura, parseo y clasificación completados | D-MEM-13 |
| ST-04 | PROCESSING | PROCESSING | Error en procesamiento | catch NO mueve a ERROR (BR-002) | AMB-07 |
| ST-05 | PROCESSING | PENDING | Cleanup cron | updatedAt > 10min AND retryCount <= MAX_RETRIES (3) | D-MEM-35, D-MEM-40 |
| ST-06 | PENDING | PENDING | Cleanup cron | PENDING con updatedAt > 10min → reset para reintento | D-MEM-35 |
| ST-07 | PROCESSING | ERROR | Cleanup cron | retryCount > MAX_RETRIES (3) | D-MEM-35, LIM-10 |
| ST-08 | PROCESSING | ERROR | Cleanup cron | Archivo no existe en /storage/ | AMB-04 |

---

## 4. Transiciones Prohibidas

| Desde | Hacia | Razón |
|-------|-------|-------|
| IMPORTED | cualquiera | Estado terminal exitoso. No se reabre ni se reimporta. |
| ERROR | cualquiera | Estado terminal de fallo. Requiere intervención manual o nuevo import. |
| PROCESSING | ERROR (via import) | Solo cleanup cron puede mover a ERROR (BR-002). El import NUNCA mueve directamente a ERROR. |
| PENDING | ERROR (directo) | PENDING siempre pasa por PROCESSING antes de poder fallar. |
| PENDING | IMPORTED (directo) | Siempre pasa por PROCESSING. |

---

## 5. Regla de Idempotencia en Transiciones

| Situación | Resultado |
|-----------|-----------|
| Import de conversación nueva | → PENDING → PROCESSING → IMPORTED |
| Import de conversación existente (IMPORTED) | → ALREADY_INDEXED (sin cambio de estado) |
| Import de conversación existente (PROCESSING) | → ALREADY_INDEXED (sin cambio de estado, cleanup maneja) |
| Import de conversación existente (ERROR) | → Nuevo intento: si mismo externalSessionId, ALREADY_INDEXED |
| Import incremental VTT_CHANNEL | → UPDATED (sin cambio de status, permanece IMPORTED) |

---

## 6. Campos de Estado en BD

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `statusId` | UUID (FK) | Referencia a ConversationStatusCatalog |
| `retryCount` | Int | Contador de reintentos del cleanup (default: 0) |
| `errorMessage` | String? | Mensaje descriptivo del error (null si no hay error) |
| `updatedAt` | DateTime | Última modificación — usado por cleanup para detectar STALE |
| `importedAt` | DateTime? | Timestamp de cuando se completó el import (null hasta IMPORTED) |

**Nota:** Cleanup filtra por `statusId` directo (no por join con `status.code`) para performance (DB-OBS-02).
