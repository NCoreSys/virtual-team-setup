## 1. Resolución de Ambigüedades UX

Antes de los diagramas, se documentan las decisiones UX que aplican transversalmente a los flujos.

| AMB | Decisión | Justificación |
|-----|----------|---------------|
| **AMB-UX-01** | POST /upload muestra spinner "Procesando..." con auto-poll cada 30s al estado de la conversación. Si tras 3 minutos sigue en PROCESSING, muestra "El sistema reintentará automáticamente. Puedes cerrar esta ventana." | El cleanup cron reintenta cada 5 min (BR-006). El usuario no necesita esperar indefinidamente. 3 min = 6 polls, suficiente para el happy path (<10s). |
| **AMB-UX-02** | ALREADY_INDEXED se muestra como banner informativo (azul) con texto "Esta conversación ya fue importada" y link "Ver conversación" que navega al Conversation Viewer. No es error. | Reimportar no es fallo del usuario — es idempotencia funcionando (BR-001). Rojo sería confuso. |
| **AMB-UX-03** | Health parcial muestra semáforos individuales por componente (BD, Storage, Redis). Verde = OK, Amarillo = SLOW/DEGRADED, Rojo = DOWN/UNREACHABLE. El estado global es el peor de los tres. | El Admin necesita diagnosticar qué componente falla sin revisar logs (US-030). |
| **AMB-UX-04** | Dashboard muestra "Agentes activos (últimos 30 días)" con la ventana temporal visible en la UI. Coordinado con SA: BR-015 define 30 días fijos, no configurable en R1. | Consistente con AMB-UC-05 (resuelta por PM) y BR-015. El usuario debe saber qué ventana se usa. |

**Comportamiento técnico:**
- El contenido completo se lee del filesystem, no de BD (D-MEM-43)

- Archivos JSONL grandes pueden tardar — mostrar skeleton + spinner para el contenido

### UF-07 · Manual Upload

```mermaid
flowchart TD
    A[Usuario navega a Upload] --> B[Mostrar formulario de upload]
    
    B --> FORM["Formulario:<br/>1. Seleccionar archivo .json<br/>2. Fuente: auto-detectar / CLAUDE_WEB / CHATGPT<br/>3. Botón: Subir conversación"]
    
    FORM -->|Submit| C["POST /upload con archivo"]
    
    C -->|201 IMPORTED| SUCCESS["Banner éxito (verde):<br/>Conversación importada exitosamente<br/>Botones: Ver conversación / Subir otra"]
    
    C -->|"200 ALREADY_INDEXED<br/>(AMB-UX-02)"| INFO["Banner info (azul):<br/>Esta conversación ya fue importada<br/>Link: Ver conversación existente"]
    
    C -->|400 formato inválido| ERR1["Banner error (rojo):<br/>El archivo no tiene un formato reconocido.<br/>Formatos soportados: export claude.ai,<br/>export ChatGPT.<br/>Botón: Intentar con otro archivo"]
    
    C -->|400 archivo vacío/corrupto| ERR2["Banner error (rojo):<br/>El archivo está vacío o no es JSON válido<br/>Botón: Intentar con otro archivo"]
    
    C -->|"Respuesta indica PROCESSING<br/>(AMB-UX-01)"| POLL["Spinner: Procesando...<br/>Auto-poll cada 30s"]
    
    POLL -->|Poll: IMPORTED| SUCCESS
    POLL -->|"Poll: aún PROCESSING<br/>(después de 3 min)"| TIMEOUT["Info:<br/>El sistema reintentará automáticamente.<br/>Puedes cerrar esta ventana.<br/>Botón: Ver en lista de conversaciones"]
    POLL -->|Poll: ERROR| ERR3["Error:<br/>El procesamiento falló después de<br/>múltiples intentos.<br/>Botón: Reintentar upload"]
    
    SUCCESS -->|Ver conversación| NAV1["→ Conversation Viewer (UF-03)"]
    SUCCESS -->|Subir otra| B
    INFO -->|Ver existente| NAV1
    TIMEOUT -->|Ver lista| NAV2["→ Conversations List (UF-02)"]
```

**Endpoint:** `POST /api/conversations/upload`  
**Actor:** Cualquier rol (A4)  
**UC:** UC-007 | **US:** US-006 | **BR:** BR-001, BR-002, BR-019  
**Auth:** Ninguna (R1)

**Flujo de estados post-upload:**
1. POST /upload → respuesta inmediata con conversationId y status
2. Si status = IMPORTED → éxito inmediato
3. Si status = PROCESSING → UI inicia auto-poll cada 30s a GET /conversations/:id
4. Si después de 6 polls (3 min) sigue PROCESSING → mostrar mensaje informativo
5. El cleanup cron (BR-006) reintentará cada 5 min en background

### UF-08 · Health Status

```mermaid
flowchart TD
    A[Admin accede a Health Status] --> B["GET /health"]
    
    B -->|"200 status=ok"| OK["Estado: HEALTHY ●<br/>BD (PostgreSQL): ● OK<br/>Storage (/storage/): ● OK<br/>Redis: ● OK"]
    
    B -->|"200 status=degraded<br/>(AMB-UX-03)"| DEG["Estado: DEGRADED ⚠<br/>Semáforos individuales por componente:<br/>● OK / ⚠ SLOW / ✕ DOWN"]
    
    B -->|"503 status=down"| DOWN["Estado: DOWN ✕<br/>Semáforos con componentes fallidos<br/>Mensaje: Servicio no disponible"]
    
    B -->|Error de conexión| NOCONN["Error:<br/>No se puede conectar con el servicio<br/>Botón: Reintentar"]
    
    OK --> REFRESH["Auto-refresh cada 60s<br/>o botón Refresh manual"]
    DEG --> REFRESH
    DOWN --> REFRESH
    REFRESH --> B
```

**Endpoint:** `GET /api/health`  
**Actor:** Admin / DO (A5)  
**UC:** UC-019 | **US:** US-029, US-030  
**Auth:** Ninguna (R1)

**Semáforos (AMB-UX-03):**
- ● Verde = OK
- ⚠ Amarillo = SLOW / DEGRADED
- ✕ Rojo = DOWN / UNREACHABLE
- Estado global = el peor de los tres componentes
