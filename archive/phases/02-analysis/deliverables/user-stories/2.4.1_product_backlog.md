## Criterio de Priorización

El ranking dentro de cada categoría MoSCoW se basa en:

1. **Dependencias técnicas:** US que desbloquean otras van primero (US-001 desbloquea todo el flujo de consulta)
2. **Valor para consumidores principales:** Runtime y PB son consumidores contractuales — sus US van antes que las de UI User
3. **Riesgo técnico:** US con SLA contractual (<500ms) van temprano para validar la viabilidad
4. **Volumen de uso esperado:** Import automático (US-001) se ejecutará miles de veces; upload manual (US-006) decenas
