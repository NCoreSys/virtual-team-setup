## EP-01 · Import & Storage (10 stories)

### US-001 · Import sesión CLAUDE_SDK

| Campo | Valor |
|-------|-------|
| **Épica** | EP-01 |
| **Actor** | Runtime v1.1 |
| **Prioridad** | Must |
| **RFs** | RF-001, RF-003, RF-004, RF-005, RF-008, RF-010 |
| **UCs** | UC-003 |

**Como** Runtime v1.1, **quiero** importar una sesión JSONL de tipo CLAUDE_SDK con `externalSessionId = {run_id}:r{N}:{agentRole}`, **para** persistir la conversación del agente al terminar cada ronda del orquestador.

**Notas:** El import valida con Zod, escribe JSONL en `/storage/CLAUDE_SDK/`, parsea turns/blocks, guarda `contentPreview` (500 chars), ejecuta clasificación determinística y transiciona a IMPORTED. Ref: ADDENDUM §2, D-INT-01..04.

### US-002 · Import idempotente (ALREADY_INDEXED)

| Campo | Valor |
|-------|-------|
| **Épica** | EP-01 |
| **Actor** | Runtime v1.1 |
| **Prioridad** | Must |
| **RFs** | RF-002 |
| **UCs** | UC-005 |

### US-005 · Import incremental VTT_CHANNEL (append-only)

| Campo | Valor |
|-------|-------|
| **Épica** | EP-01 |
| **Actor** | Hook Manager VTT |
| **Prioridad** | Should |
| **RFs** | RF-007 |
| **UCs** | UC-006 (flujo A) |

**Como** Hook Manager, **quiero** hacer import incremental de un canal VTT_CHANNEL que ya fue importado, **para** actualizar el canal sin borrar historial.

**Notas:** Append-only para mensajes (AgentMessage). Upsert para participantes: agrega nuevos, no modifica existentes (AMB-UC-03 resuelta, AMB-02 MS-018). Actualiza `title`/`summary` con valores del último import. Retorna `{ status: "UPDATED", newMessages: N }`. Ref: D-MEM-16.

**[TL-05]:** El import incremental SÍ actualiza participantes nuevos via upsert. Si un agente se suma a un AGENT_CLARIFICATION después del primer import, se agrega como participante.

| Campo | Valor |
|-------|-------|
| **Épica** | EP-01 |
| **Actor** | Sistema |
| **Prioridad** | Must |
| **RFs** | RF-031 |
| **UCs** | UC-020 |

## EP-02 · Memory Context (5 stories)

### US-011 · Obtener contexto en <500ms

| Campo | Valor |
|-------|-------|
| **Épica** | EP-02 |
| **Actor** | Prompt Builder v1.3 |
| **Prioridad** | Must |
| **RFs** | RF-011, RF-013, RF-014 |
| **UCs** | UC-008 |

**Como** Prompt Builder, **quiero** obtener el contexto de un agente en <500ms, **para** armar prompts con memoria histórica sin bloquear el runtime.

**Notas:** 6 queries paralelas (Promise.all): recentSessions, recentDecisions, frequentFiles, recentReviews, costSummary, projectContext. Solo conversaciones IMPORTED. Ref: D-MEM-07, ADDENDUM §4.

---

### US-012 · Timeout fail-fast 504

| Campo | Valor |
|-------|-------|
| **Épica** | EP-02 |
| **Actor** | Prompt Builder v1.3 |
| **Prioridad** | Must |
| **RFs** | RF-012 |
| **UCs** | UC-009 |

**Como** Prompt Builder, **quiero** recibir HTTP 504 MEM-ERR-504 si el contexto tarda más de 500ms, **para** manejar el error explícitamente y ensamblar el prompt sin contexto.

**Notas:** NO retry, NO degraded mode, NO cache fallback. Promise.race con timeout 500ms. Ref: D-MEM-07, D-MEM-37.
