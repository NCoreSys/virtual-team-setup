## Estimación por Story

### EP-01 · Import & Storage

| US | Historia | SP | Justificación |
|----|----------|-----|---------------|
| US-001 | Import CLAUDE_SDK | **5** | Flujo multi-paso: validación Zod → idempotencia → create → write storage → parseo turns/blocks → clasificación → transition IMPORTED. Incluye resolución de catálogos desde Maps. |
| US-002 | Import idempotente | **3** | Lógica condicional (findUnique + retorno ALREADY_INDEXED) pero reutilizable en todos los imports. |
| US-003 | Race condition P2002 | **3** | Captura de error Prisma específico + fallback a findUnique. Requiere testing concurrente. |
| US-004 | Import VTT_CHANNEL | **5** | Similar a US-001 pero con participants[], messages[], storage en /_reviews/ y catálogos adicionales (messageType, messageStatus, priority). |
| US-005 | Import incremental | **5** | Lógica de upsert participantes + append mensajes + actualización campos. Requiere detectar si es primer import o incremental. |
| US-006 | Upload manual | **3** | Reutiliza flujo de import pero con format detection (claude.ai vs ChatGPT) y adapter. |
| US-007 | Import CLI | **3** | Adapter específico para CLAUDE_CLI pero flujo base compartido con US-001. |
| US-008 | Import CLAUDE_WEB | **2** | Adapter simple: parsea formato export de claude.ai. |
| US-009 | Import ChatGPT | **2** | Adapter simple: parsea formato export de ChatGPT. |
| US-010 | Cleanup reintenta fallidos | **5** | Cron job con lógica de selección (statusId + updatedAt), retry counting, manejo de archivo faltante y transición a ERROR. |

**Subtotal EP-01:** 36 SP

### EP-02 · Memory Context

| US | Historia | SP | Justificación |
|----|----------|-----|---------------|
| US-011 | Contexto <500ms | **8** | 6 queries paralelas (Promise.all), cada una con joins específicos, filtros por IMPORTED, índices compuestos. La más compleja del sistema. |
| US-012 | Timeout fail-fast 504 | **3** | Promise.race con timer 500ms + retorno de error estructurado. Requiere testing de timing. |
| US-013 | Filtrar por proyecto | **2** | WHERE clause adicional en queries de contexto. projectId siempre obligatorio (D-MEM-39). |
| US-014 | Filtrar por tarea | **2** | WHERE clause adicional opcional. |
| US-015 | Contexto síncrono | **1** | Diseño inherente — no hay implementación adicional. Documentar que no se soporta polling. |

**Subtotal EP-02:** 16 SP

### EP-04 · Cost Tracking

| US | Historia | SP | Justificación |
|----|----------|-----|---------------|
| US-022 | Costo por proyecto | **3** | Aggregation query (SUM tokens, SUM cost) con JOINs a Conversation + filtros from/to. |
| US-023 | Desglose por agente | **3** | GROUP BY primaryAgentId dentro del aggregation de proyecto. |
| US-024 | Costo de agente | **3** | Similar a US-022 pero filtrado por agentId con breakdown por proyecto y modelo. |
| US-025 | USD reales | **2** | Mapeo directo de ConversationUsage.totalCostUsd. Sin recalcular. |
| US-026 | Solo SDK con costo | **2** | Filtro en query para excluir conversaciones sin costo o marcar null explícitamente en response. |
| US-027 | Trend semanal | **5** | Requiere SQL raw con `DATE_TRUNC('week', "startedAt")` porque Prisma groupBy no soporta agrupación temporal nativa (TL-08). |

**Subtotal EP-04:** 18 SP
